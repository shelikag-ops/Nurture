Setup Guide

Get Sarah's Dashboard live in ~15 minutes. Two parts: Firebase (database) + GitHub Pages (hosting).

Part 1: Firebase Setup ~8 min

Firebase gives us a free real-time database. When Sarah checks off a task on her iPad, you'll see it update on your laptop instantly.

1 Create a Firebase project

  1. Go to console.firebase.google.com
  2. Click "Add project" (or "Create a project")
  3. Name it something like sarah-homeschool
  4. You can disable Google Analytics (not needed) — click Continue
  5. Wait for the project to be created, then click Continue

2 Add a Web App

  1. On the project home page, click the web icon (</>) to add a web app
  2. Give it a nickname like sarah-dashboard
  3. Do NOT check "Firebase Hosting" (we're using GitHub Pages instead)
  4. Click Register app
  5. You'll see a config block like this — copy it:
const firebaseConfig = {
  apiKey: "AIzaSy.....................",
  authDomain: "sarah-homeschool.firebaseapp.com",
  databaseURL: "https://sarah-homeschool-default-rtdb.firebaseio.com",
  projectId: "sarah-homeschool",
  storageBucket: "sarah-homeschool.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef123456"
};
Paste your config here and I'll generate the updated file for you: 

    


    
3 Enable the Realtime Database

    
    
      
  1. In the Firebase Console sidebar, click Build → Realtime Database
    2. 
      
  2. Click "Create Database"
    3. 
      
  3. Choose the location closest to you (e.g., us-central1 if in US, asia-southeast1 if in India)
    4. 
      
  4. Select "Start in test mode" (we'll secure it properly in a moment)
    5. 
      
  5. Click Enable
    6. 
    


    

      Important: "Test mode" allows anyone to read/write for 30 days. We'll add proper security rules in the next step so only your app can access the data.
    


    
4 Set Security Rules

    
    
      
  1. In the Realtime Database section, click the Rules tab
    2. 
      
  2. Replace the default rules with this:
    3. 
    


    
{
  "rules": {
    "homeschool": {
      ".read": true,
      ".write": true,
      "shared": {
        ".read": true,
        ".write": true
      },
      "sarah": {
        ".read": true,
        ".write": true
      },
      "parent": {
        ".read": true,
        ".write": true
      }
    }
  }
}


    

      Why these rules? Since we're using PIN-based auth (not Firebase Auth), we keep the rules open but scoped to just the /homeschool path. The data isn't sensitive (it's homework scores, not financial data), and the app URL isn't publicly listed. For extra security later, we can add Firebase Auth and lock rules to authenticated users only.
    


    
Click Publish.


    
5 Update your config file

    
    
      
  1. Open js/firebase-config.js in any text editor
    2. 
      
  2. Replace the FIREBASE_CONFIG block with your actual values from Step 2
    3. 
      
  3. Save the file
    4. 
    


    

      Firebase is done! The app will now sync data across all devices. Open index.html in your browser — you should see "Firebase Sync: Connected" on the dashboard.
    


    

    
Part 2: GitHub Pages ~7 min


    
This puts your app on a URL that Sarah can bookmark on her iPad.


    
1 Create a GitHub repository

    
    
      
  1. Go to github.com/new
    2. 
      
  2. Name it whatever you chose (e.g., sarah-learning or homeschool-app)
    3. 
      
  3. Set it to Private (your daughter's data doesn't need to be public)
    4. 
      
  4. Click Create repository
    5. 
    


    

      Note: GitHub Pages works with private repos on the Free plan as of 2024. The site will still be accessible via URL, but the code stays private.
    


    
2 Push your files

    
Open Terminal on your Mac, navigate to the project folder, and run:


    
cd "path/to/Sarah Homeschooling"

git init
git add index.html dashboard.html setup-guide.html js/ Corrections/
git commit -m "Initial deploy: dashboard with Firebase sync"

git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main


    

      Replace YOUR_USERNAME and YOUR_REPO with your actual GitHub username and the repo name you chose.
    


    
3 Enable GitHub Pages

    
    
      
  1. Go to your repo on GitHub → Settings tab
    2. 
      
  2. In the left sidebar, click Pages
    3. 
      
  3. Under "Source", select Deploy from a branch
    4. 
      
  4. Choose main branch and / (root) folder
    5. 
      
  5. Click Save
    6. 
    


    
After 1-2 minutes, your site will be live at:

    
https://YOUR_USERNAME.github.io/YOUR_REPO/


    
4 Bookmark on Sarah's iPad

    
    
      
  1. Open Safari on the iPad and go to the URL above
    2. 
      
  2. Tap the Share button → Add to Home Screen
    3. 
      
  3. Name it "My Dashboard" or whatever Sarah likes
    4. 
      
  4. It now appears as an app icon on her home screen
    5. 
    


    

      You're live! Sarah opens the app on her iPad, enters her PIN (default: 1234), and everything she does syncs to your laptop in real time.

How to Update the App

    
Whenever you (or the other chat window) update dashboard.html or any file, push it to GitHub and the live site updates automatically:


    
cd "path/to/Sarah Homeschooling"
git add -A
git commit -m "Update dashboard"
git push


    
Changes go live within 1-2 minutes.
